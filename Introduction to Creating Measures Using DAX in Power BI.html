Definisi Data Analysis Expressions (DAX)

Data Analysis Expressions (DAX) merupakan kumpulan fungsi, operator, dan konstanta yang dapat digunakan dalam rumus, atau ekspresi, untuk menghitung dan menampilkan satu atau lebih nilai yang digunakan dalam Power BI, Analysis Services, dan Power Pivot di Excel.

DAX memungkinkan pengguna untuk melakukan perhitungan dan query tingkat lanjut pada data dalam tabel dan kolom terkait dalam model data tabular. DAX adalah alat yang kuat untuk analisis dan manipulasi data, dan dapat digunakan untuk memecahkan berbagai masalah perhitungan dan analisis data dasar. Data Analysis Expressions (DAX) memberikan kesempatan pada pengguna untuk menghasilkan perhitungan yang kompleks.

Poin Penting Data Analysis Expression (DAX)

DAX bukan merupakan bahasa pemrograman, melainkan bahasa rumus yang digunakan untuk menentukan penghitungan khusus untuk kolom dan calculated measure (juga dikenal sebagai calculated fields). DAX menyertakan beberapa fungsi yang digunakan dalam rumus Excel, dan fungsi tambahan yang dirancang untuk bekerja dengan data relasional dan melakukan agregasi dinamis. Berikut adalah beberapa poin dalam DAX yang perlu diperhatikan:

  1. DAX adalah bahasa fungsional (functional language) yang memiliki kode lengkap selalu berupa fungsi. Ekspresi DAX yang dapat dieksekusi mungkin berisi pernyataan kondisional, fungsi bertumpuk (nested function), referensi nilai, dan lain-lain.
  2. Rumus DAX memiliki dua tipe data primer yaitu numerik dan non-numerik. Tipe data numerik meliputi bilangan bulat, desimal, mata uang, dll. Sedangkan tipe data non-numerik terdiri dari string dan objek biner.
  3. Ekspresi DAX dievaluasi dari fungsi terdalam hingga fungsi terluar terakhir. Hal ini mengakibatkan penulisan rumus yang benar pada DAX menjadi penting.

Pada DAX pengguna dapat menggunakan nilai tipe data campuran sebagai input dalam rumus DAX dan konversi akan berlangsung secara otomatis selama eksekusi rumus. Nilai output akan dikonversi menjadi tipe data yang diinstruksikan untuk rumus DAX.

Konsep Data Analysis Expression (DAX)

DAX (Data Analysis Expressions) telah menjadi bahasa formula yang penting dalam dunia bisnis dan analisis data. DAX digunakan dalam berbagai platform analisis data, termasuk Microsoft Power BI, Power Pivot, dan Analysis Services Tabular Mode. Berikut adalah konsep fundamental dalam DAX yang digunakan dalam analisis data pada Power BI:

  1. Data Analysis Expressions (DAX) Formula − Syntax

    2. Syntax merupakan seluruh elemen yang dapat digunakan untuk membangun sebuah formula kalkulasi atau dapat disimpulkan bahwa syntax merupakan cara penulisan atau bagaimana formula akan ditulis. Beberapa elemen syntax DAX yang perlu diperhatikan adalah sebagai berikut:

    1. Nama Measure
    2. Equals Operator
    3. DAX Functions
    4. Parenthesis
    5. Reference Table
    6. Reference Column

    Contoh penerapan dari syntax DAX adalah sebagai berikut:

    "Total Sales = SUM(Sales[SalesAmount])"

    Syntax di atas bertujuan untuk menghitung total penjualan dengan cara menambahkan setiap nilai dari kolom “SalesAmount” dari tabel “Sales”.

  2. Data Analysis Expressions (DAX) Calculation Types

    3. DAX formula dapat disebut sebagai kalkulasi karena menghitung nilai input dan mengembalikan nilai yang dihasilkan. Dua tipe ekspresi atau kalkulasi DAX dalam Power BI adalah:

    1. Calculated Columns: membuat kolom baru di tabel yang sudah ada, digunakan ketika akan membuat kolom dengan informasi yang difilter atau diurutkan.
    2. Calculated Measures: menciptakan bidang yang memiliki nilai agregat seperti jumlah, rasio, persentase, rata-rata, dan lain-lain.
  3. Data Analysis Expressions (DAX) Functions

    4. Fungsi DAX adalah rumus yang menghitung nilai dalam argumen dapat berupa kolom, angka, teks, atau lainnya. Setiap fungsi DAX melakukan operasi tertentu pada nilai tersebut. Beberapa poin penting fungsi DAX adalah:

    1. Fungsi DAX selalu merujuk pada seluruh kolom/bidang atau tabel, dan tidak akan pernah merujuk pada nilai individu.
    2. Fungsi DAX memberikan fleksibilitas untuk membuat formula yang diterapkan secara baris per baris.
    3. Dalam beberapa kasus, fungsi DAX mengeluarkan output tabel lengkap (full tabel) yang dapat digunakan dalam formula DAX lain yang membutuhkan satu set himpunan nilai.
    4. Fungsi DAX memiliki kategori yang dikenal sebagai fungsi time intelligence seperti fungsi yang digunakan untuk menghitung waktu.
  4. Data Analysis Expressions (DAX) Context

    5. Konsep konteks DAX penting dalam memperoleh pemahaman lengkap tentang bahasa DAX dan cara kerja pada Power BI. Berikut dua jenis konteks DAX:

    1. Row context: memperhitungkan baris tertentu yang telah difilter dalam ekspresi DAX. Row context berfokus pada pengoperasian rumus pada baris saat ini. Jenis konteks ini paling sering diterapkan pada ukuran baris.
    2. Filter context: formula atau DAX expression berfokus kepada value spesifik di dalam sebuah baris. Filter Context diaplikasikan ketika menggunakan functions seperti CALCULATE, FILTER, RELATED, ALL, dan sebagainya.

    Tools Dalam Data Analysis Expressions (DAX)

    DAX dapat membantu memperoleh hasil data yang maksimal. Mempelajari berbagai tools seperti rumus dan ekspresi DAX dibutuhkan untuk meningkatkan pemahaman. Berikut adalah tools yang umum digunakan dalam DAX:

    1. FILTER

      2. Filter digunakan untuk mengembalikan subset dari tabel atau ekspresi. Dalam laporan Power BI, filter dapat diterapkan pada keseluruhan laporan, halaman tertentu, atau visual tertentu. Berikut syntax dari tool Filter:

      Filter(<table>,<filter>)

    2. ALL

      3. Tool related berguna untuk membantu mengembalikan nilai terkait dari tabel lain. Tool related digunakan untuk mengembalikan subset atau menghapus filter apa pun pada tabel atau kolom.

      RELATED(<column>

    3. TOTALYTD/TOTALQTD/TOTALMTD

      4. Tool ini merupakan jenis tool data analysis expressions untuk memanipulasi data menggunakan periode waktu. Periode waktu dapat berupa hari, bulan, kuartal, dan tahun.

      TOTALYTD(<expression>,<dates>[,<>filter>][,<year_end_date>])

    4. CALCULATE

      5. Tool CALCULATE digunakan untuk mengevaluasi ekspresi dalam konteks yang dimodifikasi oleh filter tertentu. Tool ini dapat membuat semua tabulasi sekaligus untuk menyelesaikan beberapa perhitungan.

      CALCULATE(<expression>,<filter1>,<filter2>...)

    Membangun Measure Menggunakan DAX

    Sebelum memulai implementasi DAX pada Power BI, unduh data source Superstore terlebih dahulu. Data Superstore merupakan salah satu open dataset memuat informasi tentang penjualan dan keuntungan Superstore.

    Untuk menghubungkan dataset yang telah diunduh sebelumnya ke dalam Power BI dapat menggunakan langkah-langkah seperti berikut:

    1. Buka program Power BI Desktop yang telah terinstal.
    2. Pada program yang terbuka pilih “Excel workbook” di bagian “Select a data source or start with a blank report”, karena akan digunakan format file Excel untuk dataset seperti ilustrasi di bawah ini.
    3. Setelah memilih tautan "Microsoft Excel", jendela "Open" akan terbuka, pilih lembar kerja/file yang sesuai untuk digunakan dalam Power BI seperti ilustrasi berikut ini.
    4. Setelah file terhubung pada Power BI, akan muncul Window Navigator seperti ilustrasi di bawah untuk memilih dataset yang akan digunakan.
    5. Pilih dataset yang akan digunakan pada window Navigator, lalu pilih “Load”, dan akan muncul Window Processing Queries yang menampilkan status transformasi dan menghubungkan data seperti ilustrasi di bawah ini.

    Pada course ini dicontohkan untuk membangun rata-rata penjualan (Sales) dari tabel orders. Untuk membangun measure menggunakan DAX dari dataset yang telah diunduh sebelumnya ke dalam Power BI dapat menggunakan langkah-langkah berikut:

    1. Pilih “New measure” pada panel “Home”.
    2. Pada panel “Measure tools” yang terbuka ketikkan.

      3. "Average Sales=AVERAGE(Order[Sales])"

      Kemudian pilih ikon centang seperti pada ilustrasi di bawah.

    3. Measure atau ukuran baru yang terbentuk akan tampil di sisi kanan tableau pada panel “Data” seperti pada ilustrasi di bawah.
    4. Untuk memastikan apakah measure sudah sesuai akan dicontohkan untuk membangun tabel antara “Average Sales” dengan “Category”. Lakukan drag-and-drop “Average Sales” dan “Category” pada panel “Values” seperti ilustrasi di bawah.
    5. Kemudian akan terbentuk tabel seperti ilustrasi di bawah.

    Membangun Kalkulasi Kolom Menggunakan DAX

    Pada course ini akan dicontohkan untuk membangun kalkulasi kolom menggunakan DAX dari dataset yang telah diunduh sebelumnya. Berikut adalah langkah-langkah membangun kalkulasi kolom:

    1. Pilih Tabel View pada sisi kiri Power BI seperti ilustrasi di bawah, tampilan Power BI akan berubah menampilkan data yang kita gunakan.
    2. Pilih “New columns” pada panel “Tabel tools” yang terbuka seperti ilustrasi di bawah.
    3. Pada panel yang terbuka ketikkan

      4. CustomerType = VAR TotalSales = SUMX(Orders, [Sales]*[Quantity]) RETURN IF (TotalSales > 2000, "Gold", "Silver")

      Formula di atas akan memunculkan kolom bernama CustomerType dengan input apabila TotalSales lebih dari 2000 akan memiliki nilai Gold dan Silver untuk lainnya. TotalSales didapatkan dari perkalian Sales dan Quantity dari tabel order. Perintah VAR digunakan untuk membangun variabel baru agar dapat digunakan pada kalkulasi selanjutnya.

      Ketik ENTER pada keyboard lalu akan muncul kolom CustomerType pada data seperti ilustrasi di bawah.

    Membangun Tabel Kalkulasi Menggunakan DAX

    Pada course ini akan dicontohkan untuk membangun tabel kalkulasi menggunakan DAX dari dataset yang telah diunduh sebelumnya. Berikut adalah langkah-langkah membangun tabel kalkulasi:

    1. Pilih Tabel View pada sisi kiri tableau seperti ilustrasi di bawah, tampilan Power BI akan berubah menampilkan data yang kita gunakan.
    2. Pilih “New table” pada panel “Tabel tools” yang terbuka seperti ilustrasi di bawah.
    3. Pada panel yang terbuka ketikkan.

      4. SuperStore Newell= FILTER (Orders, Orders[Product Name] = “Newell 317”)

      Formula di atas akan memunculkan tabel bernama “SuperStore Newell” yang berisikan data Orders yang hanya mempunyai nilai “Newell 317” pada kolom “Product Name”.

      Ketik ENTER pada keyboard lalu akan muncul tabel Superstore Newell pada data.
