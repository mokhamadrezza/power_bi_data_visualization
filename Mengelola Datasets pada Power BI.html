Mengenal Gateway pada Power BI

Power BI memfasilitasi pengguna untuk melakukan transfer data baik antar lokal maupun beberapa layanan pada cloud seperti Microsoft Azure dengan menggunakan Gateway. Gateway Power BI adalah perangkat lunak yang berfungsi sebagai jembatan untuk menyediakan transfer data yang cepat dan aman antara data on-premises dan beberapa layanan cloud Microsoft, termasuk Power BI, PowerApps, Power Automate, Azure Analysis Services, dan Azure Logic Apps.

Gateway memfasilitasi komunikasi cepat di balik layar yang mengalir dari satu pengguna di cloud ke sumber data lokal pengguna lain lalu kembali lagi ke cloud.

Jenis Gateway pada Power BI

Gateway Power BI secara umum terbagi menjadi dua tipe berdasarkan mode penggunaan, yaitu mode organisasi dan mode personal. Berikut adalah perbedaan dasar antara dua mode gateway Power BI:

  1. Mode organisasi: Memungkinkan beberapa pengguna untuk terhubung ke beberapa sumber data lokal dan cocok untuk skenario kompleks.
  2. Mode pribadi: Memungkinkan satu pengguna untuk terhubung ke sumber data. Jenis gateway ini hanya dapat digunakan dengan Power BI dan tidak dapat dibagikan dengan pengguna lain.

Cara Kerja Gateway pada Power BI

Gateway Power BI bekerja dibalik layar saat pengguna bekerja di cloud dan berinteraksi dengan elemen yang tersambung ke sumber data lokal, sehingga pengguna tidak perlu melakukan intervensi berlebih. Berikut hal yang terjadi ketika menggunakan gateway pada Power BI:

  1. Layanan cloud membuat kueri dan kredensial terenkripsi untuk sumber data lokal. Kueri dan kredensial dikirim ke antrean gateway untuk diproses.
  2. Layanan cloud gateway menganalisis kueri dan mendorong permintaan ke Microsoft Azure Service Bus.
  3. Azure Service Bus mengirimkan permintaan yang tertunda ke gateway.
  4. Gateway mendapatkan kueri, mendekripsi kredensial, lalu menyambungkan ke satu atau beberapa sumber data dengan kredensial tersebut.
  5. Gateway mengirim kueri ke sumber data untuk dijalankan.
  6. Hasil dikirimkan dari sumber data kembali ke gateway lalu ke layanan cloud. Layanan kemudian menggunakan hasilnya.

    Mempromosikan Datasets pada Power BI Service

    Proses business intelligence melibatkan kolaborasi. Berbagi dataset merupakan cara untuk berkolaborasi dalam organisasi. Untuk memastikan bahwa organisasi memiliki data yang konsisten untuk membuat keputusan, penting untuk membuat dan berbagi himpunan data yang dioptimalkan lalu mendukung dataset tersebut sebagai salah satu sumber fakta. Power BI menyediakan dua cara untuk mendukung berbagi dataset:

    1. Promosi: Pengguna dapat mempromosikan dataset yang siap untuk penggunaan luas.
    2. Sertifikasi: Pengguna dapat meminta sertifikasi untuk dataset yang dipromosikan. Sertifikasi menjadi proses yang sangat selektif, sehingga hanya dataset yang benar-benar andal dan otoritatif yang digunakan pada organisasi.

    Membangun Workspace Power BI Service

    Sebelum mengelola dataset pada Power BI Service, diperlukan file Power BI dengan ekstensi .pbix. Unduh file IT Spend yang berisi tentang report dan dataset mengenai informasi tentang budget, pengeluaran & perkiraan (forecast) perusahaan pada teknologi dengan ilustrasi dataset seperti di bawah ini.

    Untuk mengelola dataset pada Power BI Service diperlukan workspace dengan data yang diunduh sebelumnya. Berikut adalah langkah yang dapat digunakan untuk membangun workspace:

    1. Buka Power BI Service pada link app.powerbi.com.
    2. Pada panel bar yang terletak pada sisi kiri pilih ikon
    3. Pada panel yang terbuka pilih “New workspace” untuk memilih membuat workspace baru seperti ilustrasi di bawah.
    4. Pada panel “Create a workspace” yang terbuka ketikkan nama workspace yang diinginkan pada “Name”, pastikan tidak sama dengan nama workspace yang sudah ada.
    5. Ketikkan deskripsi untuk workspace yang akan dibangun pada “Description”.
    6. Pilih Apply seperti pada ilustrasi di bawah.
    7. Halaman Power BI Service akan berganti pada workspace baru.
    8. Pilih menu “Upload” lalu pilih “Browse” seperti pada ilustrasi di bawah.
    9. Pilih file yang akan diupload pada workspace, lalu pilih “Open” seperti ilustrasi di bawah.
    10. Workspace yang dibangun sebelumnya akan berisi 3 file setelah proses upload selesai yaitu file untuk Dashboard, Dataset, dan Report seperti ilustrasi di bawah.
